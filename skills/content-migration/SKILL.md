# Content Migration

## Content Audit Process

1. **Inventory:** Catalog all content (pages, posts, assets, documents).
2. **Assess:** Score each piece on accuracy, usefulness, quality, and alignment.
3. **Decide:** Keep as-is, revise, merge, archive, or delete.
4. **Plan:** Map content to new structure.
5. **Execute:** Migrate in phases, not all at once.

## Migration Checklist
- [ ] URL redirect map (old → new)
- [ ] Image/asset inventory and migration
- [ ] SEO metadata preserved or improved
- [ ] Internal links updated
- [ ] 404 monitoring post-launch
- [ ] Analytics baseline captured before migration

## Common Mistakes
- Migrating everything (including outdated, redundant content).
- Not setting up redirects (SEO disaster).
- Losing metadata during migration.
- Not testing content in the new design before launch.
- Migrating without an editorial review.
