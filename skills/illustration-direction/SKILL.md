# Illustration Direction

## Defining an Illustration Style

### Properties to Decide
- **Complexity:** Simple/geometric vs detailed/organic.
- **Color:** Limited palette vs full spectrum. Flat vs gradient.
- **Line:** Thick/thin, round/sharp, visible/invisible.
- **Perspective:** Flat/2D vs isometric vs 3D.
- **Characters:** Abstract shapes, geometric people, realistic figures.
- **Texture:** Clean vectors vs grain/noise/hand-drawn.

### Consistency Rules
- Same stroke weight across all illustrations.
- Same color palette (derived from brand colors).
- Same level of detail (don't mix simple and complex).
- Same perspective/angle conventions.
- Character proportions consistent.

## When to Illustrate
- Empty states (guide user action).
- Onboarding (explain concepts).
- Error states (soften the experience).
- Marketing (differentiate from competitors).
- Complex concepts (make abstract ideas concrete).

## Anti-Patterns
- Using different illustration libraries in the same product.
- Stock illustrations without customization.
- Illustration that doesn't match the product's maturity level.
- Decorative illustration that doesn't communicate anything.
