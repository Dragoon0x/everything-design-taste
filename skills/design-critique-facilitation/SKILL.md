# Design Critique Facilitation

## Critique Format

### Setup (2 min)
- Designer presents context: goal, constraints, user, stage of work.
- Type of feedback requested: "I need help with the navigation structure" (not "thoughts?").

### Review (5-10 min)
- Silent review first (1-2 min). Everyone looks, no talking.
- Structured feedback: "I notice [observation], which might impact [outcome], because [reasoning]."
- Facilitator keeps feedback on-topic.

### Discussion (5-10 min)
- Designer asks follow-up questions.
- Group discusses highest-priority issues.
- Avoid redesigning in the room.

### Close (2 min)
- Designer summarizes key takeaways.
- No action items assigned in critique (designer decides what to act on).

## Rules
1. Critique the work, not the person.
2. Be specific. "The hierarchy is unclear" → "The H2 and body text are too similar in weight."
3. One person talks at a time.
4. Opinions are valid when connected to principles or user needs.
5. "I like it" is not feedback. Neither is "I don't like it."
6. Seniority doesn't determine correctness.
