# Design Sprint Facilitation

## 5-Day Design Sprint

| Day | Focus | Output |
|-----|-------|--------|
| Monday | Understand | Problem map, long-term goal, sprint questions |
| Tuesday | Diverge | Sketches, crazy 8s, solution concepts |
| Wednesday | Decide | Storyboard, prototype plan |
| Thursday | Prototype | Realistic facade |
| Friday | Test | User interviews, validated/invalidated assumptions |

## Workshop Facilitation

### Rules
1. Timeboxes are sacred. 5 minutes means 5 minutes.
2. Diverge before converging. Generate before evaluating.
3. Write before discussing. Silent ideation prevents groupthink.
4. Vote with dots. Dot voting surfaces preferences without debate.
5. Decider has final say. Someone needs authority to break ties.

### Tools
- Timer visible to all.
- Sticky notes (physical or Miro/FigJam).
- Voting dots.
- How Might We format for reframing problems.

### Common Mistakes
- Inviting too many people (5-7 max).
- Not having a decider in the room.
- Skipping the prototype (testing ideas, not opinions).
- Testing with colleagues instead of real users.
